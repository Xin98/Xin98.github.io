实验部分

模型训练所需资源对比

背景: 基于TinyTL: Reduce Memory, Not Parameters for Efficient On-Device Learning 论文

  1. tiny net 在batch size 为512 只占用5000MB的显存，而Proxyless net 则需要15000MB(两张3090的卡同时使用时，单卡占用的显存).好处有:

    • 相同资源条件下，可使用更大的bsz进行训练，使得训练速度更快.
    • 小的显存占用代表着模型更小，可以部署到资源有限的边缘设备，充分利用算力.
    • 使用tiny net可以在fine tune模式下占用更小的内存（待记录实际占用显存）

  2. 之前理解的TinyNet是将原来的block修改为lite residual block，实际上，应该是在原来的每一层的模型参数保持不变的情况下，加上一个lite residual，然后在downstream 的迁移学习中使用，而freeze原来的block，来达到降低内存，提升fine tune 过程的有效性的效果。

  3. cifar 优化不明显的原因是因为图像大小比较小

  4. resnet 没有 inverted bottleneck blocks 而 lite residual 是基于它进行的

    利用 inverted bottleneck blocks 对resnet进行改造 然后加以利用lite residual 观察效果

  5. 论文的部分组织结构 可以为： 1. 模型压缩–nas + 迁移学习 + lite residual 2. 模型压缩网络结构剪枝，量化等 + 迁移学习

  6. lite_residual 的参数多于原来模型的原因是 group设置太小

  7. 第四章实验思路，先将训练好的模型进行量化，然后加入lite块迁移到资源受限的设备上进行训练，实验结果可以的话可以作为创新点
