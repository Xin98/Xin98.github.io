nexus-6P升级到Android-8.0的问题及解决记录

背景

最近在做一个实验室的项目，需要用到nexus6P手机，考虑到nexus6P手机的系统版本过低，所以老师要求我们将手机的系统升级到较高的版本。

nexus6P当前系统 : andriod6.0，目标系统 : android8.0。

尝试

太久没接触过手机刷机，踩了许多雷，这两天使用过NRT(Nexus Root Toolkit)和OTA(Over-The-Air Technology)安装包的sideload安装方法，均出现过问题，所以安装无法成功。由于我暂时也还没有找到这两种安装方式解决方法，这里就不贴出相关的问题了。

Platform-tools更新的问题及解决方法

首先排除工具的影响，请大家下载最新版本的工具包.

  1. 配置环境

    将platform-tools的目录加入到path系统环境变量中，例如我的platform-tools在C:\Users\G-Xin\Desktop\platformtools下，相应的系统环境变量(path)就增加一条记录。配置完成后，你可以在任意的文件夹中使用fastboot及adb命令。

  2. 准备好factory镜像

    镜像下载: 地址

    因为是采用fastboot升级安装方式，所以镜像的选择也非常重要，nexus6P的升级镜像有两种: 1. factory image 2. ota image。而fastboot升级方法只能选择使用factoryimage。

  3. 安装

    将下载的安装包解压到任意文件夹中。然后将imgxxxx.zip解压到当前文件夹中。

    这时候正常情况下，将手机启动到bootloader界面，然后双击flash-all.bat（windows下），flash-all.sh(linux下)即可完成安装。nexus6P启动到bootloader方法: 关机后，同时按电源键和音量下键即可进入。

  4. 遇到的问题

    但是事情不总是一帆风顺的，很不幸，我没能安装成功，当我执行了flash-all.bat后，出现了update package missing system.img 错误。而当我解决了这个问题，安装系统成功后，又出现了卡在google logo处无法进入系统的情况。如果大家安装时遇到了这个问题，我也非常的遗憾。但是别着急，如果你按照我下面的方法重新进行安装，就能解决它，让我们继续吧！
