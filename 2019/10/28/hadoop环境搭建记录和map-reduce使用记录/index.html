hadoop介绍

hadoop是一种基于Java语言的分布式系统架构，主要用来处理大数据相关的问题。hadoop环境搭建主要有三种模式:

  1. 单机。单机模式主要是为了方便用户调试代码，所有程序运行在一个JVM中，这使得编写map/reduce程序看起来和我们平时编写单机程序一模一样。
  2. 伪分布式。Hadoop的所有进程(NameNode, DataNode等)全部运行在一台机器上，模拟一个小的分布式集群，伪分布式模式是完全分布式模式的一个特例。
  3. 完全分布式。Hadoop的进程运行在一个集群上。

这一次主要是给大家介绍，如何用虚拟机搭建一个Hadoop的完全分布式集群。

配置

本次主要使用了4台虚拟机组成：1(master) + 3(slave) 的分布式架构

单个节点的配置:

  • 系统: CentOs6.1
  • 内存: 2G
  • 硬盘: 20G
  • JDK: 1.8.0 下载地址
  • hadoop版本: 2.7.3 下载地址

如果大家按照我的配置去搭建环境，至少需要8G以上的内存才能跑起来，所以对大家的物理计算机配置要求有点高。建议: 如果你的计算机是8G内存，请将虚拟机内存设置为1G，或者使用3台虚拟机的 1 + 2 架构。

      通过fs.default.name指定NameNode的IP地址和端口号，通过hadoop.tmp.dir指定hadoop数据存储的临时文件夹。

      vi core-site.xml

      特别注意：如没有配置hadoop.tmp.dir参数，此时系统默认的临时目录为：/tmp/hadoo-hadoop。而这个目录在每次重启后都会被删除，必须重新执行format才行，否则会出错。

    3. hdfs-site.xml的配置

      通过dfs.replication指定HDFS的备份因子为3，通过dfs.name.dir指定namenode节点的文件存储目录，通过dfs.data.dir指定datanode节点的文件存储目录。

集群启动

  1. 格式化HDFS文件系统

    第一次启动hadoop时，首先需要格式化hdfs，如果没有配置core-site.xml中的hadoop.tmp.dir属性，那么每次启动都需要此操作。

    hadoop namenode -format

  2. 启动hadoop

    首先切换到hadoop目录

    cd $HADOOP_HOME

    • 旧版本

      sbin/start-all.sh

    • 新版本(hadoop2.0)

      sbin/start-yarn.sh

  3. 关闭hadoop

    首先切换到hadoop目录

    cd $HADOOP_HOME

    • 旧版本

      sbin/stop-all.sh

    • 新版本(hadoop2.0)

      sbin/stop-yarn.sh

  4. 状态查看

    使用jps命令查看hadoop进程运行情况

    jps

    查看hadoop集群状态

    hadoop dfsadmin -report
