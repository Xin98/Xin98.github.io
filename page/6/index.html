hadoop介绍

hadoop是一种基于Java语言的分布式系统架构，主要用来处理大数据相关的问题。hadoop环境搭建主要有三种模式:

  1. 单机。单机模式主要是为了方便用户调试代码，所有程序运行在一个JVM中，这使得编写map/reduce程序看起来和我们平时编写单机程序一模一样。
  2. 伪分布式。Hadoop的所有进程(NameNode, DataNode等)全部运行在一台机器上，模拟一个小的分布式集群，伪分布式模式是完全分布式模式的一个特例。
  3. 完全分布式。Hadoop的进程运行在一个集群上。

这一次主要是给大家介绍，如何用虚拟机搭建一个Hadoop的完全分布式集群。

配置

本次主要使用了4台虚拟机组成：1(master) + 3(slave) 的分布式架构

单个节点的配置:

  • 系统: CentOs6.1
  • 内存: 2G
  • 硬盘: 20G
  • JDK: 1.8.0 下载地址
  • hadoop版本: 2.7.3 下载地址

如果大家按照我的配置去搭建环境，至少需要8G以上的内存才能跑起来，所以对大家的物理计算机配置要求有点高。建议: 如果你的计算机是8G内存，请将虚拟机内存设置为1G，或者使用3台虚拟机的 1 + 2 架构。

集群启动

  1. 格式化HDFS文件系统

    第一次启动hadoop时，首先需要格式化hdfs，如果没有配置core-site.xml中的hadoop.tmp.dir属性，那么每次启动都需要此操作。

    hadoop namenode -format

  2. 启动hadoop

    首先切换到hadoop目录

    cd $HADOOP_HOME

    • 旧版本

      sbin/start-all.sh

    • 新版本(hadoop2.0)

      sbin/start-yarn.sh

  3. 关闭hadoop

    首先切换到hadoop目录

    cd $HADOOP_HOME

    • 旧版本

      sbin/stop-all.sh

    • 新版本(hadoop2.0)

      sbin/stop-yarn.sh

  4. 状态查看

    使用jps命令查看hadoop进程运行情况

    jps

    查看hadoop集群状态

    hadoop dfsadmin -report

问题记录

  1. 在使用ssh的时候，我不幸出现过 connect refused 的错误，不知道大家有没有出现过。有可能是ssh服务没打开，或者没有安装。

    要解决这个问题，可以尝试执行以下命令

    查看ssh服务的状态

    service ssdh status

    若出现unrecognized service说明ssh服务未安装。这时我们需要安装open-ssh

    yum -y install openssh-server

    然后启动ssh服务

    service sshd start
