XML学习笔记(2) – XML语法的DTD的验证方法

最后修改时间: 2019-10-16 20:27:00

1. XML 的正确规则

  • XML 文档必须要有根标签
  • XML 文档必须要有关闭标签
  • XML 文档的标签必须要正确的嵌套
  • XML 文档的属性值必须要用引号括起来
  • XML 文档的标签大小写必须一致

2. XML 的 DTD 验证方法

DTD (Document Type Definition) 是文档定义类型，它指定了一个 XML 文档标签格式应该遵循规范。它可以直接被包含在 XML 文档中，也可以作为 XML 的外部引用。

DTD是 XML 文档结构的规范，所以首先要了解 XML 文档的构成 :

  1. 元素，即标签和其包含的其他信息。元素可包含其他元素，文本，或空内容。
  2. 属性，即元素的附加信息，存在于开始标签中，以key=”value”对的形式存在。
  3. PCDATA (Parsed Character Data) 即可被解析的字符数据，位于元素的开始标签后，关闭标签前。
  4. CDATA 即普通字符数据，不可被解析，和PCDATA位置一样。
  5. 实体，参考笔记1。

知道了构建模块，接下来的介绍当然就是 DTD 对 这些构建模块的定义方法了。

  • 元素定义

    解释 : !ELEMENT是元素定义的关键字，element-name是元素的命名定义，category是元素的类型定义，element-content是元素的内容定义，需要用括号括起来。在定义语句中category和element-content二选一。

    例如 :

  1. 空元素的定义方法 – 对应 category

    <!ELEMENT empty-element EMPTY>

    对应到xml为 : <empty-element />

    其中empty-element为元素名，EMPTY为category关键字。

  2. 带有数据的元素定义 – 对应 (element-content)

    对应到xml为 : <data-element>存放着"PCDATA"类型或者"CDATA"类型的数据</data-element>

  3. 带有任何内容的元素定义 – 对应 category

    <!ELEMENT any-element ANY>

  4. 带有子元素的元素定义 – 对应 (element-content)

    • 一般定义 :

    • 特殊定义 (类似于正则通配符的规则) :

      1. 选定的子元素至少出现一次，使用+符号来描述子元素

        <!ELEMENT element-name (subElementName+)>

        例如 :

      2. 选定的元素出现一次或者零次，使用?符号来描述子元素

        <!ELEMENT element-name (subElementName?)>

      3. 选定的元素出现零次或者多次，使用*符号来描述子元素

        <!ELEMENT element-name (subElementName*)>

      4. 选定的多个元素可任意选择的出现一次

        <!ELEMENT element-name (subElementName1|subElementName2)> <!-- subElementName1和subElementName2任意出现一次 -->

      5. 以上四种规则均可进行组合使用，称为申明混合型

        例如 :

        <!ELEMENT element-name ((subElementName1|subElementName2)?)>

        说明 : subElementName1和subElementName2出现一次或者零次

  • 属性定义

    <!ATTLIST element-name attribute-name attribute-type defualt-attribute-value>

    例如 :

    DTD : <!ATTLIST node title CDATA "just a title">

    XML : <node title = "my DTD"></node>

    1. 属性类型 attribute-type

      类型 描述 举例
      CDATA 属性是字符数据类型 <xiaoMing age = “18”>
      (emu1,emu2,…) 属性值取于此枚举列表 枚举列表:(male|female)
      gender是xiaoMing的枚举类型属性
      <xiaoMing gender = “female1”>X
      <xiaoMing gender = “male”>√
      ID 属性值为唯一的ID(任意两个元素不允许出现相同的id) xiaoMing,xiaoLi的id属性为ID类型
      <xiaoMing id = “abc”>
      <xiaoLi id = “abc”> X
      <xiaoLi id = “abd”> √
      IDREF 属性值为其他元素的ID引用(即值引用元素相等) xiaoMing,xiaoLi,xiaoHua的id属性分别为ID，ID，IDREF类型
      <xiaoMing id = “abc”>
      <xiaoLi id = “abd”>
      <xiaoHua id = “abc”> √
      <xiaoHua id = “abd”> √
      IDREFS 属性值为一个或多个其他元素的ID的引用 xiaoMing,xiaoLi,xiaoHua的id属性分别为ID，ID，IDREFS类型
      <xiaoMing id = “abc”>
      <xiaoLi id = “abd”>
      <xiaoHua id = “abc abd”> √
      NMTOKEN 属性值为xml的限定名(即满足xml命名规范的值) <xiaoMing email = “bvcx sd”>X
      <xiaoMing email = “bvcx_sd”>√
      NMTOKENS 属性值为xml的限定名的列表，限定名之间以空格隔开 <xiaoMing email = “bvcx sd”>√
      <xiaoMing email = “bvcx_sd”>√
      ENTITY 属性值是一个实体 使用方法与IDREF类似，首先得有ENTITY对象才能使用ENTITY属性
      ENTITIES 属性值是一个或多个实体 与IDREFS类似

    2. 属性默认值 defualt-attribute-value

      值 描述 举例
      值 属性的默认值 <!ATTLIST xiaoMing age CDATA “10”>
      #REQUIRED 属性的值是必须的 <!ATTLIST xiaoMing age CDATA #REQUIRED>
      #IMPLIED 属性的值是非必须的 <!ATTLIST xiaoMing age CDATA #IMPLIED>
      #FIXED value 属性的值是固定值value <!ATTLIST xiaoMing gender CDATA #FIX “male”>
